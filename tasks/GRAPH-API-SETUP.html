Suite Manager Portal

Microsoft Graph Email — Setup Guide

This authorizes the portal to send email through Suite Manager's Microsoft 365. Work through the eight steps below. You'll end with four values — three you hand back, one (the secret) you load yourself.

  • Configured on Suite Manager's M365 — the portal sends email as Suite Manager, not as you.
  • Requires a Global Administrator on Suite Manager's M365. If that isn't you, Chris grants the role temporarily or you screen-share with their M365 admin.
  • Pick a from-address — a dedicated mailbox like portal@suitemanagerllc.com (a shared mailbox is fine, no paid license needed).

The setup

1

Open the admin center

Go to entra.microsoft.com and sign in with the Suite Manager Global Admin account. (This is the Microsoft Entra admin center — "Entra ID" is the new name for Azure AD.)

2

Register the app

Left sidebar: Identity → Applications → App registrations → + New registration

  • Name: Suite Manager Portal — Mail
  • Supported account types: Accounts in this organizational directory only
  • Redirect URI: leave blank
  • Click Register
3

Copy two IDs

On the app's Overview page, copy and save:

  • Application (client) ID
  • Directory (tenant) ID
4

Grant the email permission

App's left sidebar: API permissions → + Add a permission → Microsoft Graph → Application permissions

Search Mail.Send, check it, click Add permissions.

Then click Grant admin consent for [Suite Manager] → Yes. The Mail.Send row must show a green Granted check — if it doesn't, email won't be authorized.

5

Create a client secret

App's left sidebar: Certificates & secrets → Client secrets → + New client secret

  • Description: Portal mail
  • Expires: 24 months (the maximum)
  • Click Add
Copy the Value immediately — the column labeled Value, not Secret ID. It is shown once and never again. Navigate away and you'll have to delete it and make a new one. Also set a calendar reminder ~22 months out to rotate it — when it expires, email silently stops.
6

Confirm the sender mailbox

Confirm your from-address (e.g. portal@suitemanagerllc.com) exists as a licensed mailbox at admin.microsoft.com. If it doesn't, create it — a shared mailbox works and needs no paid license.

7

Hand it back

ValueSensitiveAction
Directory (tenant) IDNoPaste it to Claude
Application (client) IDNoPaste it to Claude
Sender mailbox addressNoPaste it to Claude
Client secret ValueYesSet it yourself — see below
Set the secret: Cloudflare dashboard → Workers & Pages → app-suitemanagerllc → Settings → Variables and secrets → Add → Type Secret → Name GRAPH_CLIENT_SECRET → paste the Value → Save. One field, two minutes.
8

Lock the app to one mailbox — recommended, optional

By default the Mail.Send permission lets the app send as any mailbox in the tenant. Restrict it to just the portal mailbox via Exchange Online PowerShell:

Connect-ExchangeOnline
New-ApplicationAccessPolicy -AppId <client-id> `
  -PolicyScopeGroupId portal@suitemanagerllc.com `
  -AccessRight RestrictAccess `
  -Description "Restrict portal app to the portal mailbox"

Optional for getting email working, but worth doing before real volume. Claude can walk you through it separately.

What happens next

Once Claude has the three non-secret values and the secret is set in Cloudflare:

  • src/lib/email/graph.ts — token exchange + Graph sendMail
  • Invite emails — the login link auto-sends on every new user
  • Approval-request email to admins (the link Chris clicks)
  • Upload notifications to Strand
Roughly one focused session once the credentials exist.
